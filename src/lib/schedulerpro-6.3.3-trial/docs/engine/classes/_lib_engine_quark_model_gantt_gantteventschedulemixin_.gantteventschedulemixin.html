shouldPushingDepAffectScheduling

  • shouldPushingDepAffectScheduling(dependency: InstanceType<this["project"]["dependencyModelClass"]>): CalculationIterator<boolean>

  • Inherited from ConstrainedLateScheduleMixin.shouldPushingDepAffectScheduling

    • Defined in lib/Engine/quark/model/scheduler_pro/ConstrainedByDependenciesEventScheduleMixin.ts:36

    The method defines whether the provided dependency (called pushing dependency) should constrain the successor/predecessor (in forward/backward direction) or not. If the method returns true the dependency constrains the event being pushed and does not do that when false returned. By default the method returns true if the dependency is active and if this event is active (or both the pushing event and the event being pushed are inactive).

    This is used when calculating startDateConstraintIntervals.

    Parameters

    • dependency: InstanceType<this["project"]["dependencyModelClass"]>

      Dependency to consider.

    Returns CalculationIterator<boolean>

    true if the dependency should push, false if not.

shouldRollupChildEndDate

  • shouldRollupChildEndDate(child: ConstrainedEarlyEventMixin): CalculationIterator<boolean>

  • Inherited from HasDateConstraintScheduleMixin.shouldRollupChildEndDate

    • Defined in lib/Engine/quark/model/scheduler_pro/ConstrainedScheduleMixin.ts:440

    The method defines whether the provided child event should be taken into account when calculating this summary event earlyEndDate. Child events roll up their earlyEndDate values to their summary tasks. So a summary task earlyEndDate gets equal to its maximal child earlyEndDate.

    If the method returns true the child event is taken into account and if the method returns false it's not. By default, the method returns true to include all child events data.

    Parameters

    • child: ConstrainedEarlyEventMixin

      Child event to consider.

    Returns CalculationIterator<boolean>

    true if the provided event should be taken into account, false if not.
